## Overview

The OlaCarsBackend is a monolithic Node.js and Express RESTful API application providing comprehensive management operations for a car sales and maintenance business. The application manages everything from users, organizational hierarchy (country managers, admins, branch managers, staff), branches, financial operations (ledgers, payments, accounting codes, taxes), purchases, vehicles, and external suppliers.

## Components

The application code resides inside `Src/modules/` with 17 standalone components. Each component typically contains:
* `Model/`: Mongoose schemas and virtuals defining the MongoDB collection.
* `Repo/`: Database abstraction layer handling direct mongoose queries and logic.
* `Controller/`: Request/Response handling, payload validation, and HTTP logic.
* `Routes/`: Definition of endpoints and attaching Swagger documentation and Middleware (Auth).

### Key Modules:

- **Branch / BranchManager**: Organizational structure for branch data.
- **Admin / OperationAdmin / FinanceAdmin / CountryManager**: Hierarchical multi-tiered role access management.
- **User**: End-user logic and authentication.

## Data Flow

1. Client sends HTTP REST request to `/api/[module]`.
2. Request hits the respective `[Module]Router.js` and usually passes through `authentication`/`authorization` middleware validating JWT tokens.
3. The Router passes the `req` object to the `[Module]Controller.js` which extracts parameters.
4. The Controller invokes operations strictly within `[Module]Repo.js`. 
5. The Repo queries or updates the database via Mongoose Models (`[Module]Model.js`) and returns the document.
6. The Controller generates a standardized JSON response format.
